# Requisitos do Projeto

O sistema desenvolvido para a AMG Confecções visa atender às principais demandas operacionais da empresa, com foco em rastreabilidade, controle de pedidos e análise estratégica. Os requisitos foram definidos a partir da observação do fluxo atual da empresa e da necessidade de digitalizar processos manuais.

## Requisitos Funcionais

- Permitir o **cadastro de clientes, produtos, insumos e fornecedores**
- Realizar o **cadastro e acompanhamento de pedidos**
- Registrar o consumo de insumos por produto
- Permitir a **edição, exclusão e consulta** dos dados cadastrados (CRUD completo)
- Gerar relatórios com dados de produção, pedidos e consumo de insumos
- Exportar dados para integração com ferramentas de BI (Power BI)
- Oferecer controle de etapas do processo produtivo (modelagem, corte, costura, etc.)

## Requisitos Não Funcionais

- O sistema deve ser de **uso interno**, acessado por gestores e funcionários
- A interface deve ser **simples e intuitiva**, com foco em usabilidade
- O sistema deve garantir **persistência e integridade dos dados**
- A solução deve utilizar tecnologias de **código aberto**, como React e PostgreSQL
- Os dados devem estar disponíveis para análise em tempo real ou via relatórios atualizados regularmente
- O sistema deve estar estruturado para permitir **futuras melhorias e escalabilidade**

## Requisitos do Projeto Integrador

Como parte das exigências do Projeto Integrador I do curso de Big Data para Negócios, este projeto contempla obrigatoriamente os seguintes pontos:

1. **Uma interface com operações CRUD**
2. **Um banco de dados relacional estruturado**
3. **Relatórios e visualizações criadas no Power BI**

Esses requisitos foram totalmente incorporados à proposta, com o objetivo de garantir a coerência entre a solução desenvolvida e os critérios acadêmicos.
